Welcome to the xrd_simulator¶

This is a package for simulating X-ray Difraction from polycrystals

  • Defines polycrystals as a mesh of tetrahedral single crystals.

  • Simulates diffraction based on structure factors and a discretized detector array.

Installation¶

Anaconda¶

contomo is installable via anaconda:

conda install xrd_simulator

pip¶

as well as via pip:

pip install xrd_simulator

source¶

you may also try and build form the source. In that case first:

git clone ...

next, go to the just downloaded directory:

cd xrd_simulator

and install:

python setup.py build install

class xrd_simulator.beam.Beam(beam_vertices, wavelength, k1, k2)¶

Bases: object

Represents a monochromatic X-ray beam as a convex polyhedra.

The Beam object stores a state of an X-ray beam. In a parametric scan intervall the beam is allowed to take on wavevectors in the fan formed by [\(\boldsymbol{k}_1\), \(\boldsymbol{k}_2\)] such that all wavevectors in the scan intervall lies within the plane defined by \(\boldsymbol{k}_1\) and unto \(\boldsymbol{k}_2\). The geometry or profile of the beam is likewise restricted to rotate by the same transformation that brings \(\boldsymbol{k}_1\) unto \(\boldsymbol{k}_2\) (rodriguez rotation). I.e all vertices of the convex beam hull will rotate according to a rodriguez rotation defined by the unit vector which is in the direction of the cross product of \(\boldsymbol{k}_1\) and \(\boldsymbol{k}_2\).

Parameters

  • beam_vertices (numpy array) – Xray-beam vertices for s=0.

  • wavelength (float) – Photon wavelength in units of angstrom.

  • k1 (numpy array) – Beam wavevector for s=0 with `shape=(3,)`

  • k2 (numpy array) – Beam wavevector for s=1 with `shape=(3,)`

class xrd_simulator.detector.Detector(pixel_size, geometry_matrix)¶

Bases: object

Represents a rectangular X-ray scattering flat area detection device.

The detector can collect scattering as abstract objects and map them to frame numbers. Using a render function these abstract representation can be rendered into pixelated frames. The detector is described by a 3 x 3 geometry matrix whic columns contain vectors that attach to three of the four corners of the detector. By providing a callable geometry matrix function the detector position at any point in the scan intervall [\(\boldsymbol{k}_1, \boldsymbol{k}_2\)], \(s \in[0, 1]\) may be retrieved. This means that the detector does not need to be fixed in relation to a global laboratory coordinate system.

Parameters

  • pixel_size (float) – Pixel side length (square pixels) in units of microns.

  • geometry_matrix (callable) – geometry_matrix(s) <- G where G is a `shape=(3,3)` geometry matrix which columns attach to three corners of the detector array (units of microns).

xrd_simulator.laue.find_solutions_to_tangens_half_angle_equation(c_0, c_1, c_2, alpha)¶

Find all solutions, s, to the equation (maximum 2 solutions exists)

\[(c_2 - c_0) t^2 + 2 c_1 t + (c_0 + c_2) = 0. \quad\quad (1)\]

and .. math::alpha is the angle between k1 and k2

  • c_1 (float) – Coefficents c_0,c_1 and c_2 of equation (1).

  • c_2 (float) – Coefficents c_0,c_1 and c_2 of equation (1).

xrd_simulator.laue.get_tangens_half_angle_equation(k1, theta, G, rhat)¶

Find the coefficents of the quadratic equation

\[(c_2 - c_0) t^2 + 2 c_1 t + (c_0 + c_2) = 0. \quad\quad (1)\]

Its roots (if existing) are solutions to the Laue equations where t is

\[t = \tan(s \alpha / 2). \quad\quad (2)\]

and s is the sought parameter which parametrises all wave vectors between k1 and k2 where s=0 corresponds to k1 and s=1 to k2. Intermediate vectors are reached by rotating k1 in the plane spanned by k1 and k2.

Parameters

  • k1 (numpy array) – Beam wavevector for s=0 with `shape=(3,)`

  • theta (float) – Bragg angle in radians.

  • G (numpy array) – Sample coordinate system diffraction vector.

  • rhat (numpy array) – Unit vector orthogonal to both k1 and k2.

Returns

Coefficents c_0,c_1 and c_2 of equation (1).

Return type

(tuple of float)
